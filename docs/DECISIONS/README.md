# Architecture Decision Records

Create an ADR for a material, long-lived decision affecting architecture, persistent data, authentication/authorisation, hosting, external services, or cross-cutting developer workflow. Do not create ADRs for routine implementation details.

Name records `NNNN-short-title.md` (for example, `0001-adopt-example.md`). Valid statuses are **Proposed**, **Accepted**, **Superseded**, **Rejected**, and **Deprecated**. Do not create an ADR for this delivery-system documentation unless a real architectural choice is made.

```markdown
# NNNN: Title

- Status: Proposed
- Date: YYYY-MM-DD

## Context

## Decision

## Consequences

## Alternatives considered

## Links
```
